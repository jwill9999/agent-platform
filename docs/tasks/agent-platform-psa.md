# Task: Context window management

**Beads issue:** `agent-platform-psa`  
**Spec file:** `docs/tasks/agent-platform-psa.md` (this file)  
**Parent epic:** Context Management

## Task requirements

Implement context window management so long conversations don't exhaust LLM token limits or cause excessive cost. Add a sliding window with configurable strategy: truncation (drop oldest messages) or summarisation (LLM-generated summary of dropped context).

### Current state

- **No truncation or windowing** — `listMessagesBySession()` loads ALL messages every request
- Messages ordered by `createdAtMs ASC` (oldest first)
- `buildConversationMessages()` prepends system prompt + all history + new user message
- Token tracking exists in `HarnessState` (`totalTokensUsed`), but only for output — not input context
- `ExecutionLimits` has `maxTokens` but it's checked after LLM response, not before context construction
- Risk: large sessions blow through token limits; expensive calls; potential LLM API errors

### Target state

- **Context budget**: Configurable max tokens for input context (default: 80% of model's context window)
- **Sliding window**: Keep system prompt + most recent N messages that fit within budget
- **Summarisation (optional)**: Replace dropped messages with a short LLM-generated summary
- **Strategy selection**: Configurable per-agent: `'truncate'` (default) or `'summarize'`
- **Token counting**: Use `tiktoken` or model-specific tokenizer for accurate counts
- **Metadata**: Track `contextTokensUsed` and `messagesDropped` in state/trace

### Strategy details

**Truncation (default)**:

1. Always keep: system prompt + last user message
2. Fill remaining budget with most recent messages (newest first)
3. If even one assistant message doesn't fit, return only system + user

**Summarisation**:

1. Same as truncation, but dropped messages are replaced with a summary
2. Summary message: `{ role: 'system', content: 'Summary of prior conversation: ...' }`
3. Summary generated by a cheap/fast model call (configurable)
4. Summary cached per session (only regenerate when new messages drop off)

## Implementation plan

### Step 1: Add token counting utility

**File:** `packages/harness/src/tokenCount.ts` (new)

```typescript
export interface TokenCounter {
  count(text: string): number;
  countMessages(messages: ChatMessage[]): number;
}

// Default: approximate counter (4 chars ≈ 1 token)
export function createApproximateCounter(): TokenCounter;

// Optional: tiktoken-based counter for accuracy
export function createTiktokenCounter(model: string): TokenCounter;
```

Start with approximate counter; tiktoken as opt-in upgrade.

### Step 2: Add context window config to agent schema

**File:** `packages/contracts/src/schemas/agent.ts`

```typescript
contextWindow: z.object({
  maxInputTokens: z.number().int().positive().default(8000),
  strategy: z.enum(['truncate', 'summarize']).default('truncate'),
}).default({}),
```

### Step 3: Create context builder with windowing

**File:** `packages/harness/src/contextBuilder.ts` (new)

```typescript
export function buildWindowedContext(
  systemPrompt: string,
  history: ChatMessage[],
  newMessage: ChatMessage,
  config: ContextWindowConfig,
  counter: TokenCounter,
): { messages: ChatMessage[]; dropped: number; contextTokens: number };
```

Algorithm:

1. Count tokens for system prompt + new user message (required)
2. Iterate history from newest to oldest, adding messages while within budget
3. If strategy is 'summarize' and messages were dropped, prepend summary
4. Return selected messages in chronological order

### Step 4: Integrate into chat router

**File:** `apps/api/src/infrastructure/http/v1/chatRouter.ts`

Replace current `buildConversationMessages()` to use `buildWindowedContext()`:

```typescript
const { messages, dropped, contextTokens } = buildWindowedContext(
  systemPrompt,
  priorMessages,
  { role: 'user', content: newMessage },
  agentCtx.agent.contextWindow,
  counter,
);
```

### Step 5: Add summarisation support (optional path)

**File:** `packages/harness/src/contextSummarizer.ts` (new)

```typescript
export async function summarizeMessages(
  messages: ChatMessage[],
  model: LanguageModel,
): Promise<string>;
```

- Uses a cheap model call to summarize dropped messages
- Summary cached in session metadata (avoid re-summarizing on every request)
- Cache invalidated when new messages are dropped

### Step 6: Add trace events and state tracking

**File:** `packages/harness/src/trace.ts`

Add `context_window` event:

```typescript
{ type: 'context_window', contextTokens: number, messagesTotal: number, messagesIncluded: number, strategy: string }
```
